## 3. 평가 핵심: 강점 & 약점

### ✅ 강점 (살릴 것)
1. **외부 인터넷 가능** — SheetJS CDN 활용 가능, 설치 부담 없음
2. **Claude 활용 습관** — AI에 적극적으로 물어보는 태도로 학습 곡선 완화 가능
3. **명확한 발표 목표** — 무엇을 해야 하는지 알고 있고, 유연한 범위 조정 가능

### ⚠️ 약점 (해결 필요)
1. **개발 경험 전무 + 거의 시간 없음** — 3주는 학습과 구현을 동시에 해야 하는 타이트한 일정
2. **새로운 기술 여러 개** — SheetJS (엑셀 파싱) + 가능하면 DSX (XML 파싱) 동시 학습 필요
3. **범위 부명확** — 원래 SPEC에는 SMS + 엑셀만 있었으나, DSX 파일 요구사항 추가로 인한 혼동

## 4. 개선안: 범위 & 일정 재조정

### 개선 내용

#### 4.1 범위: Phase 1 + Phase 1.5로 명확화

**Phase 1 (필수, 7/15 발표까지)**
- ✅ SMS 오류 메시지 파싱
- ✅ 엑셀 파일에서 배치 정보 조회 (GROUP_ID/RUN_ID/SEQ_ID 검색)
- ✅ 결과 테이블 표시 (7개 정보 컬럼)
- ✅ 오류 처리 및 초기화 기능

**→ 이것만으로도 "작동하는 작품" 완성 가능**

---

**Phase 1.5 (추가, DSX 파일 검증 후 판단)**
- ⚙️ DSX 파일에서 배치 선후행 관계 추출 (텍스트 리스트)

**포함 조건:**
- Claude 검증 결과 "충분히 간단" → **포함** ✅
- Claude 검증 결과 "너무 복잡" → **건너뜀** (Phase 1만으로도 OK)

**→ 둘 다 허용, 개발자가 선택 가능**

---

#### 4.2 일정: 3주 마일스톤

| 주차 | 목표 | 예상 작업 시간 | 누적 |
|------|------|---|---|
| **Week 1** | SMS 파싱 + HTML UI 기초 | 7-9시간 | 7-9h |
| **Week 2** | SheetJS 엑셀 읽기 + 검색 + 테이블 표시 | 7-10시간 | 14-19h |
| **Week 3** | 오류 처리 + 테스트 + DSX 검증 | 5-7시간 | 19-26h |

**필요 시간 vs 가용 시간:**
- Phase 1 필요: **16-22시간**
- 3주 가용: **27-33시간**
- **여유: 5-17시간** ✅ (예상치 못한 문제 대응 가능)

---

#### 4.3 작업 분해 (Phase 1 상세)

**Week 1 (7-9시간)**
- Day 1-2: HTML/JS 기초 학습 + 환경 구성 (Claude 튜토리얼)
- Day 3: SMS 파싱 로직 구현 (정규식, 오류 처리)
- Day 4-5: 좌우 분할 레이아웃 + 파싱 결과 표시

**Week 2 (7-10시간)**
- Day 6-7: SheetJS로 엑셀 파일 읽기 + 헤더 유연 매칭
- Day 8-9: 검색 로직 (GROUP_ID/RUN_ID/SEQ_ID 우선순위)
- Day 10: 결과 테이블 표시 (7개 컬럼)

**Week 3 (5-7시간)**
- Day 14-16: 오류 처리 메시지 추가 + 버튼 상태 관리
- Day 17-19: 통합 테스트 (SMS 파싱 실패/성공, 매칭 실패/성공 케이스)
- Day 20-21: DSX 파일 검증 (Claude와 함께) 및 최종 정리

## 6. 주의사항 & 조언

### ⚠️ 성공을 위한 필수 조건
1. **Claude와의 적극적 협업**
   - 막힐 때마다 물어보기 (시간 낭비 방지)
   - 코드 리뷰 + 개선 제안 받기

2. **주말 시간 활용**
   - 주중 1시간으로는 집중이 어려움
   - 주말에 연속 2-3시간 집중 작업 필요

3. **초기 2주가 중요**
   - Week 1-2에서 기초(HTML/JS/SheetJS) 확실히 익히기
   - Week 3에서 여유 있게 테스트 + 최적화 가능

4. **범위 관리**
   - Phase 1 완주에 집중
   - Phase 1.5는 "시간 있으면 하는" 느낌으로 접근
   - 새로운 기능 추가는 절대 금지
