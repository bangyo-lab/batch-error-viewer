# 배치 오류 정보 조회 웹앱 (MVP Spec - IndexedDB)

**작성일**: 2026-06-16  
**최종 수정**: 2026-06-24  
**버전**: v4.0 (IndexedDB 기반 단순화)  
**프로젝트 규모**: 3주 (주중 1시간/일 + 주말 2-3시간/일)  
**예상 소요 시간**: Phase 1 = 12-18시간 (SheetJS 제거로 4시간 단축), Phase 1.5 = +2-3시간 (선택)

---

## 1. 문제

배치 자동화 시스템에서 오류가 발생할 때, 수신한 **SMS 오류 메시지만으로는** 어떤 배치가 실패했는지, 실패한 배치의 상세 정보(job ID, 타겟 테이블 등)를 빠르게 파악하기 어렵다.

---

## 2. 사용자 (페르소나)

- **배치 운영담당자** / 개발팀
- 오류 알림을 받고 5분 내에 해당 배치의 상세 정보를 확인해야 함
- 복붙 한 번으로 빠르게 정보를 찾고 싶음
- **반복 사용**: 같은 배치 정보 DB로 여러 오류를 빠르게 검색

---

## 3. 핵심 기능

### 3.1 Phase 1 (필수, 7/15 발표까지)

**Phase 1만으로도 "작동하는 작품" 완성 가능. 이것이 최소 목표.**

#### 3.1.1 오류 메시지 파싱
- 사용자가 SMS 오류 메시지를 텍스트 영역에 **복붙**
- **[파싱] 버튼 클릭** → 자동으로 다음 정보를 **파싱**:
  - **seq id** (예: `seq_sfpdmg502`)
  - **GROUP ID** (예: `SDF770`)
  - **RUN ID** (예: `SDFPD771`)
  - **job명** (예: `FW1차_오늘농사_수출입실적날씨월통계(16일)`)
- **파싱 형식**: 고정된 SMS 형식 (예: `[배치자동화]작업에러[260616][01:10][SDF770][SDFPD771][FW1차_...]`)
- **오류 메시지 형식**:
  ```
  [배치자동화]작업에러[YYMMDD][HH:MM][GROUP_ID][RUN_ID][job명]...
  ```

##### 3.1.1.1 파싱 실패 시
- **구체적인 오류 메시지**: `"GROUP_ID를 찾을 수 없습니다."` 또는 `"RUN_ID를 찾을 수 없습니다."`
- 사용자가 SMS 메시지 형식을 다시 확인하도록 유도

#### 3.1.2 배치 정보 DB 조회
- **초기 데이터**: 엑셀 → JSON으로 변환하여 앱에 내장 (4.1 참조)
- **앱 시작 시**: JSON 데이터를 IndexedDB에 로드 (한 번)
- 이후는 모두 IndexedDB에서 **빠른 조회**
- 파싱된 정보(GROUP ID, RUN ID, SEQ ID)를 **키로 검색**

##### 3.1.2.1 검색 로직 (우선순위)
1. **GROUP_ID**로 매칭되는 행 찾기
2. GROUP_ID가 없으면 **RUN_ID**로 검색
3. RUN_ID가 없으면 **SEQ_ID**로 검색
4. **매칭 결과**: 해당 행의 job id, 타겟테이블, 타겟테이블명 추출

##### 3.1.2.2 배치 정보 DB 구조 (IndexedDB)
```javascript
// 테이블: batches
{
  groupId: "SDF770",          // 인덱스
  runId: "SDFPD771",          // 인덱스
  seqId: "seq_sfpdmg502",     // 인덱스
  jobName: "FW1차_오늘농사_...",
  jobId: "JOB001",
  targetTable: "TB_EXP",
  targetTableName: "수출입"
}
```

- **인덱싱**: groupId, runId, seqId로 빠른 검색 가능
- **저장 용량**: 수십만 건의 배치 정보 저장 가능

##### 3.1.2.3 검색 결과 없음
- **일치하는 행이 없을 때**: `"GROUP_ID SDF770, RUN_ID SDFPD771과 일치하는 데이터가 없습니다."`
- 사용자가 SMS 정보를 다시 확인하도록 유도

#### 3.1.3 결과 표시
- **레이아웃**: 좌우 분할
  - **왼쪽**: SMS 입력 + [파싱] 버튼 + 파싱 결과
  - **오른쪽**: [검색] 버튼 + 검색 결과 테이블

##### 3.1.3.1 파싱 결과 (왼쪽)
- **읽기 전용**으로 표시 (사용자 수정 불가)
  ```
  GROUP ID: SDF770
  RUN ID: SDFPD771
  seq ID: seq_sfpdmg502
  job명: FW1차_오늘농사_수출입실적날씨월통계(16일)
  ```

##### 3.1.3.2 검색 결과 테이블 (오른쪽)
- **테이블 형식**으로 표시 (여러 행이 매칭되면 모두 표시)
- **표시 정보**: 파싱된 정보 + DB 조회 정보 총 7개
  ```
  GROUP_ID | RUN_ID | SEQ_ID | job명 | JOB_ID | 타겟테이블 | 타겟테이블명
  SDF770   | SDFPD771 | seq_sfpdmg502 | FW1차_... | JOB001 | TB_EXP | 수출입
  ```
- **여러 행 일치 시**: 모든 매칭 행을 표 형태로 표시 (운영자가 선택 가능하도록)

#### 3.1.4 사용자 액션 & 상태 초기화
- **[초기화] 버튼**: SMS 입력창만 비우기
  - 다음 오류 메시지를 빠르게 검색할 수 있도록 지원
- **파일 선택 없음**: 데이터는 앱에 이미 내장됨

## 4. 초기 데이터 준비

### 4.1 엑셀 → JSON 변환 (한 번만, 개발 전)

**목표**: 엑셀 파일의 배치 정보를 JSON으로 변환하여 앱에 내장

**방법:**
1. 엑셀 파일을 `data.json` 형태로 변환
   ```json
   [
     {
       "groupId": "SDF770",
       "runId": "SDFPD771",
       "seqId": "seq_sfpdmg502",
       "jobName": "FW1차_오늘농사_수출입실적날씨월통계(16일)",
       "jobId": "JOB001",
       "targetTable": "TB_EXP",
       "targetTableName": "수출입"
     },
     {
       "groupId": "SDF770",
       "runId": "SDFPD772",
       ...
     }
   ]
   ```

2. `data.json` 파일을 프로젝트에 저장

**도구:**
- 엑셀 → JSON 자동 변환 사이트 사용 (검색: "excel to json online")
- 또는 수동으로 엑셀 데이터를 JSON 형태로 입력

**시간**: 30분 ~ 1시간 (엑셀 데이터 크기에 따라)

### 4.2 JSON 데이터를 IndexedDB에 로드

**앱 시작 시 자동으로 실행:**
```javascript
// app.js
async function initializeDB() {
  const db = new Dexie('BatchDB');
  db.version(1).stores({
    batches: '++, groupId, runId, seqId'  // 인덱싱
  });
  
  // 첫 로드 시에만 JSON 데이터 로드
  const count = await db.batches.count();
  if (count === 0) {
    const data = await fetch('data.json').then(r => r.json());
    await db.batches.bulkAdd(data);
  }
  
  return db;
}
```

**결과**: 앱을 열 때 자동으로 DB 초기화 (사용자는 아무것도 할 필요 없음)

## 5. Phase 1 제외 범위 (이번엔 안 함)

- ❌ **서버 연동**: MVP는 로컬 IndexedDB만 사용 (향후 Phase 2에서 API 변경 가능)

- ❌ **권한 관리**: 로그인/권한 체크
- ❌ **상세 로그 조회**: 배치 상세 로그 표시 (추후 확장)

## 7. 성공 기준

### 데모 시나리오

**첫 번째 검색:**
1. SMS 오류 메시지 텍스트 복붙
   ```
   [배치자동화]작업에러[260616][01:10][SDF770][SDFPD771][FW1차_오늘농사_수출입실적날씨월통계(16일)]...
   ```
2. **[파싱] 버튼 클릭**
   - ✅ seq id, GROUP ID, RUN ID, job명 **자동 추출 완료**
   - ✅ 왼쪽 화면에 읽기 전용으로 표시

3. **[검색] 버튼 클릭**
   - ✅ IndexedDB에서 GROUP_ID/RUN_ID/SEQ_ID로 빠른 조회
   - ✅ 오른쪽 테이블에 결과 표시 (job id, 타겟테이블, 타겟테이블명 포함)

4. **결과 확인**
   - ✅ 7개 정보가 테이블에 명확하게 표시됨
   - ✅ 여러 행 일치 시 모두 표시

**두 번째 검색 (같은 DB):**
1. **[초기화] 버튼** → SMS 입력창만 비워짐
2. 새로운 SMS 메시지 복붙
3. [파싱] → [검색] (DB는 이미 로드됨)
   - ✅ 5분 내 빠른 검색 완료

✅ **성공**: 위 모든 단계가 정상 작동하고, 파싱 실패/매칭 실패 시 **구체적인 오류 메시지** 표시

## 10. 오류 처리 상세

### 10.1 파싱 오류

| 상황 | 오류 메시지 | 사용자 액션 |
|------|-----------|-----------|
| GROUP_ID 추출 실패 | `"GROUP_ID를 찾을 수 없습니다. SMS 메시지 형식을 다시 확인해주세요."` | SMS 메시지 재입력 |
| RUN_ID 추출 실패 | `"RUN_ID를 찾을 수 없습니다. SMS 메시지 형식을 다시 확인해주세요."` | SMS 메시지 재입력 |
| 기타 파싱 실패 | `"메시지 파싱에 실패했습니다. 형식을 확인해주세요: [배치자동화]작업에러[YYMMDD][HH:MM][GROUP_ID][RUN_ID][job명]..."` | SMS 메시지 확인 |

### 10.2 DB 오류

| 상황 | 오류 메시지 | 사용자 액션 |
|------|-----------|-----------|
| DB 로드 실패 | `"배치 정보 DB 초기화에 실패했습니다. 페이지를 새로고침해주세요."` | 브라우저 새로고침 |

## 14. 초기 데이터 준비 (개발 전 필수)

### 14.1 엑셀 → data.json 변환

**타이밍**: 개발 시작 전에 한 번만

**단계:**
1. 엑셀 파일 준비 (GROUP_ID, RUN_ID, SEQ_ID, JOB_ID, 타겟테이블 등 포함)
2. 온라인 변환 도구 사용 (검색: "excel to json") 또는 수동 입력
3. `data.json` 파일 생성 → 프로젝트 폴더에 저장

**예시 data.json:**
```json
[
  {
    "groupId": "SDF770",
    "runId": "SDFPD771",
    "seqId": "seq_sfpdmg502",
    "jobName": "FW1차_오늘농사_수출입실적날씨월통계(16일)",
    "jobId": "JOB001",
    "targetTable": "TB_EXP",
    "targetTableName": "수출입"
  },
  {
    "groupId": "SDF770",
    "runId": "SDFPD772",
    "seqId": "seq_sfpdmg503",
    "jobName": "수출입_일일_통계",
    "jobId": "JOB002",
    "targetTable": "TB_STAT",
    "targetTableName": "통계"
  }
]
```

**시간**: 30분 ~ 1시간
