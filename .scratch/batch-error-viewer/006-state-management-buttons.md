# 006: 상태 관리 & 버튼 액션

## What to build

앱의 버튼 활성화/비활성화 상태를 관리하고, 각 버튼의 액션을 구현합니다.

**버튼 상태 관리:**
- **[파싱] 버튼**: SMS 입력 있을 때만 활성 / 입력 없으면 비활성
- **[검색] 버튼**: 파싱 완료 후만 활성 / 파싱 전에는 비활성
- **[초기화] 버튼**: 항상 활성

**버튼 액션:**
- **[파싱]**: SMS 입력 텍스트 파싱 → 왼쪽에 결과 표시
- **[검색]**: 파싱된 정보로 IndexedDB 검색 → 우측 테이블 표시
- **[초기화]**: SMS 입력 텍스트만 비우기 (다른 영역은 유지)

## Acceptance criteria

- [ ] SMS 입력 감지 시 [파싱] 버튼 활성화
- [ ] SMS 입력 없을 때 [파싱] 버튼 비활성화
- [ ] 파싱 성공 후 [검색] 버튼 활성화
- [ ] 파싱 전 [검색] 버튼 비활성화
- [ ] [초기화] 버튼은 항상 활성 (초기 상태 포함)
- [ ] [초기화] 클릭 시 SMS 입력 텍스트만 비우기
- [ ] [초기화] 클릭 후에도 파싱 결과와 검색 결과는 유지
- [ ] SMS 입력이 다시 비워지면 [파싱] 버튼 자동 비활성화
- [ ] 상태 전환 시 UI 업데이트 즉시 반영
- [ ] 브라우저 개발자 도구에서 상태 변화 확인 가능
